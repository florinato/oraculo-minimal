📜 Especificación de API: Oráculo Pi App (v2.0)
1. Configuración de Conexión
Base URL: https://tu-servicio-cloud-run.a.run.app (Local: http://localhost:8080)
Endpoint: /api/pi/interpretar
Seguridad: Header X-Custom-Auth-Token (Valor desde FRONTEND_API_SECRET).
Respuesta: Streaming SSE (data: {"text": "..."}).
2. Estructura del Request (JSON)
El modelo InferenceRequest es compatible con versiones anteriores. El campo language es opcional, pero obligatorio para activar el modo multilingüe en la Pi App.
code
JSON
{
  "personality_prompt": "aislinn",   // morvan, aislinn, sybil
  "format_id": "pi_simple_5",       // pi_simple_5, pi_rapida_3, pi_sino_3, pi_sino_1
  "knowledge_base_id": "none",
  "user_question": "Will I find a romantic connection this summer?", 
  "language": "Answer the whole response strictly in ENGLISH.",
  "cards": [
    { "position": 1, "name": "El Sol", "is_reversed": false },
    { "position": 2, "name": "La Estrella", "is_reversed": false },

    { "position": 4, "name": "La Torre", "is_reversed": false },

Nota: Para la Pi App, enviar siempre is_reversed: false ya que el modelo está instruido para omitir la orientación.
3. Guía de Formatos y Layouts (Modo Pi)
Utiliza esta guía para renderizar el layout visual y asignar los textos a cada carta.
A. pi_simple_5 (Tirada en Cruz)
Ideal para análisis de situación. Estructura de 5 puntos.
Etiquetas: [C1] a [C5] y [RESUMEN].
Posiciones: 1: Pasado, 2: Futuro, 3: Mente, 4: Raíces, 5: Síntesis (Centro).
B. pi_rapida_3 (Línea Temporal)
Ideal para consultas rápidas. Disposición horizontal.
Etiquetas: [C1] a [C3] y [RESUMEN].
Posiciones: 1: Pasado, 2: Presente, 3: Futuro.
C. pi_sino_3 (La Balanza del Destino)
Para preguntas de Sí o No con matices.
Etiquetas: [C1] a [C3] y [RESUMEN].
Posiciones: 1: Viento a Favor (Pros), 2: El Muro (Contras), 3: El Veredicto (Final).
Regla: El [RESUMEN] comienza siempre con SÍ, NO o PROBABLEMENTE.
D. pi_sino_1 (El Oráculo Directo)
Respuesta instantánea de una sola carta.
Etiquetas: [C1] y [RESUMEN].
Posiciones: 1: El Veredicto Único.
4. Lógica Multilingüe (I18n)
El sistema detecta el idioma a través de la instrucción enviada en el campo language.
Configuración sugerida para el Frontend:
code
JavaScript
const langInstructions = {
  en: "Answer the whole response strictly in ENGLISH.",
  pt: "Responda a toda a resposta estritamente en PORTUGUÊS.",
  fr: "Répondez à toute la réponse strictement en FRANÇAIS.",
  es: "" // Si es vacío, el backend asume español por defecto
};
El backend concatenará esta frase al final de la pregunta del usuario dentro del prompt, asegurando que la IA cambie el idioma sin activar el validador de seguridad.
5. Procesamiento del Stream (Frontend)
Debido a que el texto llega en fragmentos (chunks) y por etiquetas, el frontend debe "limpiar" el contenido para mostrarlo en variables individuales.
Función de Parseo (Regex):
code
JavaScript
const parseTag = (fullText, tag) => {
  // Busca el contenido entre [Tag] y el siguiente [ o el final
  const regex = new RegExp(`\\[${tag}\\]([\\s\\S]*?)(?=\\[|$)`, 'i');
  const match = fullText.match(regex);
  if (match) {
    // Limpia espacios y símbolos sobrantes al inicio (: , - , *)
    return match[1].trim().replace(/^[:\s*-]+/, '');
  }
  return "";
};
Lógica de UI (React):
Acumulación: Ve sumando cada chunk de texto en un estado streamText.
Distribución: Crea variables calculadas:
const textC1 = parseTag(streamText, "C1");
const textSummary = parseTag(streamText, "RESUMEN");
Visualización:
Si textCX está vacío y el stream sigue activo -> Mostrar Skeleton/Spinner.
Si textCX tiene contenido -> Renderizar con react-markdown.
6. Notas de Implementación (Backend)
Para referencia del desarrollador, el backend monta el prompt de la siguiente manera para garantizar el éxito:
Carga el Rol y Formato desde los JSON (en español).
Limpia la pregunta y la valida con el Police System.
Une la pregunta con la instrucción de idioma: f"Pregunta: \"{user_question} {language}\"".
Añade una Regla Técnica: Do not translate the technical tags like [C1] or [RESUMEN].
Envía el "muro de texto" limpio (sin corchetes de listas) a Gemini.
