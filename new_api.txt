📜 Especificación de API: Oráculo Pi App (v2.1 - Modo Narrativo)
1. Configuración de Conexión
Base URL: https://tu-servicio-cloud-run.a.run.app
Endpoint: /api/pi/interpretar
Seguridad: Header X-Custom-Auth-Token (Valor desde FRONTEND_API_SECRET).
Respuesta: Streaming SSE (data: {"text": "..."}).
2. Estructura del Request (JSON)
Para el nuevo modelo de "Storytelling", se recomienda usar la personalidad "aura".

3. Guía de Formatos (Modelo de Storytelling)
A diferencia de la v2.0, el texto no se divide en cajas. Es un relato único donde las cartas aparecen como enlaces dentro del texto.
A. pi_simple_5 (La Cruz del Mindset): Relato fluido que integra 5 hitos (Pasado, Futuro, Mente, Raíces y Equilibrio).
B. pi_rapida_3 (El Sendero del Tiempo): Breve crónica sobre la evolución de la energía (Pasado, Presente, Futuro).
D. pi_sino_1 (El Oráculo Directo): Un solo párrafo de respuesta con veredicto final.
IMPORTANTE: Las cartas aparecen etiquetadas así: [C1]Nombre de la Carta[/C1].
4. Lógica Multilingüe (I18n)
El frontend debe enviar la instrucción de idioma en el campo language.
Inglés: "Answer the whole response strictly in ENGLISH."
Portugués: "Responda estritamente em PORTUGUÊS."
Español: "Responde estrictamente en ESPAÑOL."
5. Procesamiento del Stream en v0 (React)
v0 no debe separar el texto. Debe renderizar un párrafo continuo y sustituir las etiquetas por componentes de enlace que abran el contenido estático.
Lógica de Renderizado de Enlaces (Regex):
Dile a v0 que use este patrón para transformar el texto en tiempo real:
code
JavaScript
const renderNarrativaConLinks = (fullText) => {
  // Regex para detectar [C1]Nombre de Carta[/C1]
  const regex = /\[C(\d)\](.*?)\[\/C\d\]/g;
  
  // Dividimos el texto para encontrar las etiquetas
  const parts = fullText.split(/(\[C\d\].*?\[\/C\d\])/g);

  return parts.map((part, index) => {
    const match = part.match(/\[C(\d)\](.*?)\[\/C\d\]/);
    if (match) {
      const cardNum = match[1]; // ID de la carta (1 a 5)
      const cardName = match[2]; // Nombre de la carta (ej: El Sol)
      
      return (
        <button 
          key={index}
          onClick={() => handleOpenStaticCard(cardNum)}
          className="text-gold underline font-bold hover:opacity-80 transition-all"
        >
          {cardName}
        </button>
      );
    }
    // Si no es una etiqueta, es texto normal (Markdown)
    return <ReactMarkdown key={index}>{part}</ReactMarkdown>;
  });
};
El Cierre (Mantra):
El texto final tras la etiqueta [RES] (o [RESUMEN]) debe mostrarse destacado (ej: en itálica o en una caja de cita) como el mantra final de la sesión.
6. Notas de Implementación (Vercel / v0)
Carga Progresiva: A medida que llega el stream, el texto se irá completando. Los botones de las cartas aparecerán automáticamente en cuanto la IA cierre la etiqueta [/C1].
Mapeo de Datos: El cardNum (del 1 al 5) corresponde al índice del array cards enviado en la petición. Úsalo para buscar en tu JSON de contenido estático local.
No redundancia: No es necesario mostrar una lista de cartas aparte; el propio texto narrativo sirve de guía y los links invitan a la exploración.
Sugerencia para v0:
Pídele que use un componente de "Typography" elegante y que el botón de la carta tenga un pequeño icono de un destello o una carta al lado para que sea obvio que es clickeable.
